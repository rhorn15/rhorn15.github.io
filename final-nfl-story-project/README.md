# DANL 310 Final Project: NFL Storytelling

This project answers:

**What predicts NFL success most consistently: offense, defense, or balance?**

## Project files

- `nfl-story-project.qmd`: full narrative webpage for the project
- `nfl-dashboard.qmd`: interactive Quarto dashboard using the same data
- `app.R`: Shiny app for interactive team and season exploration
- `embed-shiny-app.qmd`: Quarto page to embed deployed Shiny app via iframe
- `manifest.json`: deployment manifest generated by `rsconnect::writeManifest()`
- `R/nfl_data_prep.R`: shared data loading and transformation pipeline
- `data/standings.csv`: cached nflverse standings dataset

## Data source

- nflverse standings: <https://github.com/nflverse/nfldata/raw/master/data/standings.csv>

## How to run locally

From this folder:

1. Render the main webpage:
   - `quarto render nfl-story-project.qmd`
2. Render the dashboard:
   - `quarto render nfl-dashboard.qmd`
3. Run the Shiny app:
   - `Rscript -e "shiny::runApp('.')"`

## Lecture 10 deployment checklist (Connect Cloud)

1. Generate/update manifest:
   - `Rscript -e "rsconnect::writeManifest(appDir='.', appPrimaryDoc='app.R')"`
2. Push files to GitHub:
   - `git add .`
   - `git commit -m "prepare deployment"`
   - `git push`
3. In Posit Connect Cloud:
   - Publish from GitHub
   - Framework: Shiny
   - Primary file: `app.R`
4. Copy the deployed app URL and paste it into `embed-shiny-app.qmd` (`YOUR_SHINY_APP_URL`).
5. Render the embed page:
   - `quarto render embed-shiny-app.qmd`
